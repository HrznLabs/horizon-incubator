## 2024-05-23 - Fix Clickjacking bypass vulnerability
**Vulnerability:** The anti-clickjacking framebusting logic in \`Verticals/ridesDAO/RidesVertical_Complete_Spec.html\` used the older pattern \`top.location = self.location;\`, which is vulnerable to being bypassed using the HTML5 \`sandbox\` attribute on iframes.
**Learning:** The legacy framebusting approach is insufficient against modern iframe sandboxing techniques. Additionally, when updating inline scripts, the corresponding Content-Security-Policy (CSP) SHA-256 hash in the \`<meta>\` tag must also be recalculated and updated, otherwise the script will be blocked by the browser.
**Prevention:** Always use the non-bypassable framebusting pattern \`window.top.location.replace(window.self.location.href);\` combined with proper \`Content-Security-Policy\` \`frame-ancestors\` directives (though \`frame-ancestors\` is not supported in \`<meta>\` tags, making robust framebusting crucial for static HTML files). When modifying inline scripts, remember to update their respective CSP hashes.
